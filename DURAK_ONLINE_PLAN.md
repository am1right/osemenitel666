# План реализации Дурак Онлайн

## Общая цель
Добавить первую многопользовательскую онлайн-игру "Дурак" в Chin Games с системой лобби, настраиваемыми правилами и ставками в Telegram Stars.

## Этапы реализации

- [x] Создать базовую страницу `durak-lobby.html` (значительно развита)
- [x] Реализовать навигацию из главной на страницу лобби

- База данных:
  - ✅ Таблица `durak_lobbies` (с полем pot для ставок)

  - ✅ `POST /api/durak/lobbies` — создать лобби (с заморозкой ставок)

  - ✅ Переход в состояние "playing" с плейсхолдером
  - ✅ Dev-инструменты симуляции событий (включая game_started)

### Этап 3: Игровая Логика (Серверная)
- Реализация правил Дурака на сервере (авторитетный сервер)
- Классы: Deck, Card, GameState
- Логика:
  - Раздача
  - Определение первого хода
  - Атака / Отбивка / Подкидывание
  - Перевод (для переводного дурака)
  - Завершение конов
  - Определение победителя
- Поддержка 2-6 игроков

### Этап 4: Игровой Клиент
- Полноценный UI для игры в Дурака
- Drag & Drop или клики для карт
- Синхронизация состояния через WebSocket
- Анимации (сброс карт, отбой и т.д.)

### Этап 5: Экономика и Ставки
- Интеграция с существующей системой Wallet
- При создании лобби со ставкой — заморозка Stars
- По завершении игры:
  - Победитель получает банк
  - 5% комиссии уходит боту (на внутренний кошелёк)
- Анти-чит: серверная валидация всех ходов

### Этап 6: Дополнительно
- История игр
- Рейтинг игроков в Дураке
- Админ-панель для управления лобби / банов
- Турниры / ежедневные события

## Технические Решения
- **Real-time**: FastAPI WebSocket (одно соединение на пользователя)
- **Game State**: Хранится на сервере, клиенты получают только то, что им положено видеть
- **Cheating protection**: Все действия валидируются на сервере
- **Betting**: Используем существующую таблицу `wallet` + новые транзакции типа "durak_bet"

## Риски
- Сложность правил Дурака (много edge-кейсов)
- Реал-тайм синхронизация при 6 игроках
- Деньги (ставки) — требует очень надёжной серверной логики
- Масштабирование (если будет много лобби)

## Приоритет
Сначала сделать playable версию без денег (0-ставочные лобби), потом добавить ставки.
